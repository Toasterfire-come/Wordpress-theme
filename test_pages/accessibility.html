Our Commitment to Accessibility

StockScan Pro is committed to ensuring digital accessibility for people with disabilities. We continually improve the user experience for everyone by applying relevant accessibility standards and guidelines.

Accessibility Standards

Our platform aims to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards. These guidelines explain how to make web content more accessible for people with disabilities, and more user-friendly for everyone.

Ongoing Efforts

  • Regular accessibility audits by third-party experts
  • Continuous testing with assistive technologies
  • User feedback integration from the disability community
  • Staff training on accessibility best practices
  • Accessibility-first approach in new feature development

Accessibility Features

Visual Accessibility

  • High Contrast Mode: Enhanced color contrast ratios exceeding WCAG AA standards
  • Font Size Controls: Adjustable text size from 100% to 200%
  • Color Alternatives: Information conveyed through shape and text, not just color
  • Focus Indicators: Clear visual focus indicators for keyboard navigation
  • Zoom Support: Full functionality maintained at up to 400% zoom

Motor Accessibility

  • Keyboard Navigation: Full site functionality accessible via keyboard only
  • Large Click Targets: Minimum 44x44 pixel touch targets
  • Timeout Extensions: Ability to extend or disable session timeouts
  • Drag & Drop Alternatives: Alternative methods for drag-and-drop actions
  • Voice Control Support: Compatible with voice navigation software

Cognitive Accessibility

  • Clear Navigation: Consistent and predictable navigation structure
  • Error Prevention: Clear form validation and error messages
  • Help Context: Contextual help and instructions throughout
  • Simple Language: Plain language used for important information
  • Progress Indicators: Clear feedback for multi-step processes

Hearing Accessibility

  • Visual Alerts: Visual alternatives to audio notifications
  • Captions: Closed captions for all video content
  • Transcripts: Text transcripts for audio content
  • Sign Language: Sign language interpretation for key videos
  • Text-Based Communication: Chat and email support options

Accessibility Testing

Our Testing Process

We employ a comprehensive testing strategy to ensure accessibility:

  • Automated Testing: Regular scans using tools like axe-core, WAVE, and Lighthouse
  • Manual Testing: Expert review using keyboard navigation and assistive technologies
  • User Testing: Real-world testing with users who have disabilities
  • Code Reviews: Accessibility-focused code reviews for all new features
  • Third-Party Audits: Annual comprehensive audits by accessibility experts

Testing Tools We Use

Automated Tools

  • axe DevTools
  • WAVE Web Accessibility Evaluator
  • Google Lighthouse
  • Pa11y command line tool
  • Colour Contrast Analyser

Manual Testing

  • Screen reader testing (NVDA, JAWS, VoiceOver)
  • Keyboard-only navigation
  • Voice control testing
  • Zoom and magnification testing
  • Color blindness simulation
